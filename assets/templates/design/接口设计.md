## 1. 接口概述

### 1.1 接口规范

{api_list}

**基础URL**: `https://api.example.com/v1`

**协议**: HTTPS

**数据格式**: JSON

**字符编码**: UTF-8

### 1.2 通用请求头

| 名称 | 必填 | 说明 | 示例 |
|------|------|------|------|
| Content-Type | 是 | 内容类型 | application/json |
| Authorization | 是 | 认证令牌 | Bearer {token} |
| X-Request-ID | 否 | 请求ID | UUID |
| X-Timestamp | 是 | 请求时间戳 | Unix时间戳 |

### 1.3 通用响应格式

**成功响应**:

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1234567890
}
```

**错误响应**:

```json
{
  "code": 400,
  "message": "参数错误",
  "error": {
    "field": "name",
    "reason": "不能为空"
  },
  "timestamp": 1234567890
}
```

### 1.4 错误码定义

| 错误码 | HTTP状态 | 说明 | 解决方案 |
|--------|----------|------|----------|
| 200 | 200 | 成功 | - |
| 400 | 400 | 请求参数错误 | 检查参数 |
| 401 | 401 | 未授权 | 重新登录 |
| 403 | 403 | 权限不足 | 联系管理员 |
| 404 | 404 | 资源不存在 | 检查资源ID |
| 409 | 409 | 资源冲突 | 检查资源状态 |
| 500 | 500 | 服务器错误 | 联系技术支持 |
| 503 | 503 | 服务不可用 | 稍后重试 |

## 2. 认证接口

### 2.1 用户登录

**接口描述**: 用户登录获取访问令牌

**请求方式**: POST

**请求URL**: `/auth/login`

**请求头**:

| 名称 | 值 |
|------|-----|
| Content-Type | application/json |

**请求体**:

```json
{
  "username": "testuser",
  "password": "password123",
  "captcha": "abc123"
}
```

**请求参数**:

| 参数 | 类型 | 必填 | 长度 | 说明 | 示例 |
|------|------|------|------|------|------|
| username | string | 是 | 5-50 | 用户名 | testuser |
| password | string | 是 | 6-50 | 密码 | password123 |
| captcha | string | 是 | 4-6 | 验证码 | abc123 |

**请求示例**:

```http
POST /auth/login
Content-Type: application/json

{
  "username": "testuser",
  "password": "password123",
  "captcha": "abc123"
}
```

**响应示例**:

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 7200,
    "user": {
      "id": "12345",
      "username": "testuser",
      "name": "测试用户",
      "roles": ["user"]
    }
  },
  "timestamp": 1234567890
}
```
