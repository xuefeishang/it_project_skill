## 3. 数据表设计

### 3.1 用户表 (users)

**表说明**: 存储用户基本信息

| 字段名 | 类型 | 长度 | 必填 | 默认值 | 约束 | 索引 | 说明 |
|--------|------|------|------|--------|------|------|------|
| id | string | 32 | 是 | - | PK | PK | 主键，UUID |
| username | string | 50 | 是 | - | UNQ | IDX | 用户名 |
| password | string | 255 | 是 | - | - | - | 密码hash |
| email | string | 100 | 是 | - | UNQ | IDX | 邮箱 |
| phone | string | 20 | 否 | - | - | IDX | 手机号 |
| status | tinyint | - | 是 | 1 | - | IDX | 状态：1-正常，0-禁用 |
| created_at | timestamp | - | 是 | now() | - | IDX | 创建时间 |
| updated_at | timestamp | - | 是 | now() on update | - | - | 更新时间 |

**索引**:

| 索引名 | 类型 | 字段 | 说明 |
|--------|------|------|------|
| PRIMARY | 主键 | id | 主键索引 |

| uk_email | 唯一 | email | 邮箱唯一 |
| idx_status | 普通 | status | 状态索引 |

## 10. 数据迁移

### 10.1 初始化脚本

```sql
-- 建表脚本
CREATE TABLE IF NOT EXISTS users (
    id VARCHAR(32) PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    email VARCHAR(100) NOT NULL UNIQUE,
    status TINYINT DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```
