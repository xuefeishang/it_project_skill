## 1. 命名规范

{coding_standards}

### 1.1 文件命名

- 使用小写字母和下划线
- 示例: `user_service.py`, `user_model.go`

### 1.2 变量命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 变量 | 小写+下划线 | `user_name` |
| 常量 | 大写+下划线 | `MAX_COUNT` |
| 函数 | 小写+下划线 | `get_user_info()` |
| 类名 | 大驼峰 | `UserService` |

### 1.3 函数命名

- 使用动词或动词短语
- 示例: `getUser()`, `saveUser()`, `deleteUser()`

## 2. 代码格式

### 2.1 缩进

- 使用空格缩进（2或4个空格）
- 禁止使用制表符

### 2.2 行长度

- 每行代码不超过120字符

### 2.3 空行

- 函数之间空2行
- 类之间空2行
- 逻辑块之间空1行

## 3. 注释规范

{documentation}

### 3.1 文件注释

```python
"""
文件描述

作者: {author}
创建日期: {date}
"""
```

### 3.2 函数注释

```python
def get_user(user_id: str) -> dict:
    """
    获取用户信息

    Args:
        user_id: 用户ID

    Returns:
        用户信息字典

    Raises:
        ValueError: 当用户ID无效时
    """
    pass
```

### 3.3 行内注释

- 注释应该解释"为什么"，而不是"是什么"
- 示例: `# 使用二分查找提高查询效率`

## 4. 代码风格

### 4.1 函数设计

- 函数长度不超过50行
- 参数不超过5个
- 单一职责原则

### 4.2 类设计

- 类职责单一
- 适当使用继承
- 避免过深的继承层次

## 5. 错误处理

| 场景 | 处理方式 |
|------|----------|
| 参数校验 | 抛出异常或返回错误码 |
| 资源获取 | 检查返回值 |
| 数据库操作 | 事务管理 |

## 6. 性能要求

{testing}

### 6.1 数据库查询

- 避免N+1查询
- 使用索引
- 限制返回数量

### 6.2 缓存使用

- 热点数据缓存
- 设置合理过期时间

## 7. 安全规范

- 输入参数验证
- SQL注入防护
- XSS防护
- 敏感数据加密

## 8. 代码审查

- 每次提交需要至少一人审查
- 审查 checklist:
  - [ ] 代码风格符合规范
  - [ ] 注释完整清晰
  - [ ] 无明显bug
  - [ ] 性能合理
