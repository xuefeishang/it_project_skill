## 通用写作原则

### 1.1 清晰性

- 使用简洁明确的语言
- 避免模糊不清的表达
- 使用专业术语时要定义

### 1.2 完整性

- 确保信息完整无遗漏
- 提供必要的上下文
- 包含所有相关细节

### 1.3 一致性

- 使用统一的术语和格式
- 保持风格一致
- 遵循命名规范

### 1.4 可读性

- 使用清晰的段落结构
- 适当使用列表和表格
- 避免过长的段落

## 章节写作指南

### 2.1 引言/背景

**目的**: 提供文档的上下文信息

**包含内容**:
- 为什么要创建这个文档
- 文档要解决什么问题
- 文档的读者是谁
- 文档的范围和限制

**写作要点**:
- 简明扼要，1-2段即可
- 说明文档的价值和目的
- 明确文档的使用场景

**示例模板**:

```markdown
## 1. 引言

### 1.1 背景

{background}

### 1.2 目的

本文档的目的是：
1. 明确项目需求
2. 指导系统设计
3. 作为开发依据

### 1.3 范围

**包含范围**:
- 用户管理功能
- 订单管理功能

**不包含范围**:
- 支付接口
- 数据分析功能

### 1.4 读者

本文档的读者包括：
- 产品经理
- 系统设计师
- 开发人员
- 测试人员
```

### 2.2 需求描述

**目的**: 详细描述系统需求

**包含内容**:
- 功能需求
- 非功能需求
- 约束条件
- 假设条件

**写作要点**:
- 使用用户故事描述功能需求
- 明确需求优先级
- 使用可验证的描述

**示例模板**:

```markdown
## 2. 需求描述

### 2.1 功能需求

#### US-001 用户登录

**作为** 用户
**我希望** 能够使用用户名和密码登录系统
**以便** 访问我的个人信息和订单

**验收标准**:
- [ ] 用户可以使用有效的用户名和密码登录
- [ ] 登录失败时显示错误提示
- [ ] 三次失败后锁定账户

### 2.2 非功能需求

#### 2.2.1 性能需求

- 系统响应时间 < 2秒
- 支持1000并发用户
- 系统可用性 > 99.9%

#### 2.2.2 安全需求

- 密码加密存储
- 使用HTTPS协议
- 实施角色权限控制
```

### 2.3 设计章节

**目的**: 描述系统的技术设计

**包含内容**:
- 系统架构
- 模块划分
- 接口设计
- 数据设计

**写作要点**:
- 使用图表辅助说明
- 描述关键设计决策
- 说明设计理由

**示例模板**:

```markdown
## 3. 系统设计

### 3.1 系统架构

{architecture}

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   前端应用   │────▶│   API网关   │────▶│   后端服务   │
└─────────────┘     └─────────────┘     └─────────────┘
                           │                    │
                           ▼                    ▼
                     ┌─────────────┐     ┌─────────────┐
                     │   缓存层    │     │   数据库    │
                     └─────────────┘     └─────────────┘
```

### 3.2 模块设计

{modules}

| 模块名称 | 功能描述 | 依赖模块 |
|----------|----------|----------|
| 用户管理 | 用户注册、登录、信息管理 | 无 |
| 订单管理 | 订单创建、查询、修改 | 用户管理、商品管理 |
```

### 2.4 测试章节

**目的**: 描述测试计划和用例

**包含内容**:
- 测试策略
- 测试范围
- 测试用例
- 测试数据

**写作要点**:
- 用例描述要清晰可执行
- 包含预期结果
- 考虑边界情况

**示例模板**:

```markdown
## 4. 测试用例

### TC-001 用户登录测试

**测试场景**: 用户使用有效凭据登录

**前置条件**:
- 用户已注册
- 用户状态正常

**测试步骤**:
1. 打开登录页面
2. 输入用户名 "testuser"
3. 输入密码 "password123"
4. 点击"登录"按钮

**预期结果**:
- 登录成功
- 跳转到首页
- 显示用户名

**实际结果**: [待填写]

**测试状态**: [通过/失败]
```

### 2.5 部署章节

**目的**: 描述系统的部署过程

**包含内容**:
- 环境要求
- 安装步骤
- 配置说明
- 验证方法

**写作要点**:
- 步骤详细可操作
- 提供命令示例
- 包含故障排查

**示例模板**:

```markdown
## 5. 部署指南

### 5.1 环境要求

**硬件要求**:
- CPU: 4核心以上
- 内存: 8GB以上
- 硬盘: 100GB以上

**软件要求**:
- 操作系统: Linux (CentOS 7+)
- 数据库: PostgreSQL 12+
- 运行环境: Node.js 16+

### 5.2 安装步骤

```bash
# 1. 克隆代码仓库
git clone https://github.com/example/project.git
cd project

# 2. 安装依赖
npm install

# 3. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填写配置信息

# 4. 初始化数据库
npm run db:migrate

# 5. 启动服务
npm start
```

### 5.3 验证安装

访问 http://localhost:3000，确认系统正常运行。
```

## 特殊章节指南

### 3.1 用户故事

**格式**: As a [角色], I want [功能], so that [价值]

**写作要点**:
- 明确用户角色
- 描述具体功能
- 说明业务价值

**示例**:
```markdown
**作为** 系统管理员
**我希望** 能够批量导入用户数据
**以便** 快速初始化系统用户
```

### 3.2 验收标准

**格式**: Given [前置条件], When [操作], Then [结果]

**写作要点**:
- 每个标准独立可测试
- 结果可验证
- 覆盖主要场景

**示例**:
```markdown
**验收标准**:

**Given** 用户已登录且订单状态为"待支付"
**When** 用户点击"取消订单"按钮
**Then** 订单状态更新为"已取消"
**And** 库存数量恢复
```

### 3.3 API接口

**格式**: 方法 + 路径 + 参数 + 响应

**写作要点**:
- 明确请求方法
- 详细描述参数
- 提供响应示例

**示例**:
```markdown
### GET /api/users/{id}

**描述**: 获取指定用户信息

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | string | 是 | 用户ID |

**请求示例**:
```http
GET /api/users/12345
Authorization: Bearer <token>
```

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "12345",
    "name": "张三",
    "email": "zhangsan@example.com"
  }
}
```
```

### 3.4 数据字典

**格式**: 字段名 + 类型 + 说明 + 约束

**写作要点**:
- 列出所有字段
- 明确数据类型
- 说明约束条件

**示例**:
```markdown
### User 表

| 字段名 | 类型 | 长度 | 允许空 | 默认值 | 说明 |
|--------|------|------|--------|--------|------|
| id | string | 32 | 否 | - | 主键，UUID |
| name | string | 50 | 否 | - | 用户名 |
| email | string | 100 | 否 | - | 邮箱地址 |
| created_at | timestamp | - | 否 | now() | 创建时间 |
```

## 常见问题处理

### 4.1 如何描述复杂流程

**方法**: 使用流程图 + 分步说明

```markdown
### 4.1 用户注册流程

**流程图**:

```
用户填写信息 → 验证信息 → 创建账户 → 发送验证邮件 → 用户激活 → 注册完成
```

**详细步骤**:

1. 用户在注册页面填写用户名、密码、邮箱
2. 系统验证信息格式和唯一性
3. 验证通过后创建账户
4. 发送验证邮件到用户邮箱
5. 用户点击验证链接激活账户
6. 注册完成，用户可登录
```

### 4.2 如何处理可选项

**方法**: 使用"可选"标记或条件描述

```markdown
### 5.1 安装步骤

**必需步骤**:
1. 下载安装包
2. 运行安装程序
3. 配置基本设置

**可选步骤**:
- [ ] 启用高级功能
- [ ] 配置监控告警
- [ ] 设置备份计划
```

### 4.3 如何处理多种情况

**方法**: 使用条件分支说明

```markdown
### 6.1 错误处理

**错误代码说明**:

| 代码 | 说明 | 处理方式 |
|------|------|----------|
| 400 | 请求参数错误 | 检查参数格式 |
| 401 | 未授权 | 重新登录 |
| 403 | 权限不足 | 联系管理员 |
| 404 | 资源不存在 | 检查资源ID |
| 500 | 服务器错误 | 联系技术支持 |
```

### 4.4 如何描述约束条件

**方法**: 单独章节列出约束

```markdown
### 7. 约束条件

#### 7.1 技术约束

- 必须使用 PostgreSQL 数据库
- 前端框架限定为 React
- API 版本使用 RESTful

#### 7.2 业务约束

- 订单金额不能超过10000元
- 用户年龄必须满18岁
- 库存不足时不能下单
```
